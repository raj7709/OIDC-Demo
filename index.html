GitHub OIDC Learning Demo

Focus on the key idea: GitHub proves its identity with a temporary token instead of stored secrets.

What is OIDC?

OpenID Connect (OIDC) is a way for an app or workflow to ask a trusted provider for proof of identity. In this demo, GitHub is the identity provider.

Simple idea: GitHub Actions can request a short-lived token and use it to access a cloud resource without saving passwords or API keys in the repo.

Simple workflow steps

Step 1

GitHub Actions starts when code is pushed or a workflow is triggered.

Step 2

GitHub issues a temporary OIDC token for this workflow run.

Step 3

The workflow uses that token to assume a cloud role, such as an AWS role, without a stored secret.

Best minimal example

This is the part students should read first. It shows the OIDC flow in the simplest form.

The sample workflow uploads a small demo file to S3 and then lists the bucket contents. This keeps the example concrete and easy to follow.

name: Simple OIDC Demo
on:
  push:
    branches: [ main ]

permissions:
  id-token: write

env:
  AWS_REGION: us-east-1
  S3_BUCKET: ${{ secrets.AWS_S3_BUCKET }}

jobs:
  demo:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Configure AWS using OIDC
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ secrets.AWS_ROLE_ARN }}
          aws-region: ${{ env.AWS_REGION }}

      - name: Upload demo file to S3
        run: |
          echo "Hello from GitHub OIDC demo" > oidc-demo.txt
          aws s3 cp oidc-demo.txt s3://${S3_BUCKET}/oidc-demo.txt

      - name: List S3 bucket contents
        run: aws s3 ls s3://${S3_BUCKET}/

The workflow above shows the core OIDC concept: GitHub sends a token, the workflow uses that token for AWS access, and no permanent secret is stored in the repository.

See the simple example file at .github/workflows/simple-oidc-demo.yml. Set AWS_ROLE_ARN and AWS_S3_BUCKET in GitHub Secrets. The AWS role should allow S3 PutObject and ListBucket.

Why this is the best demo for OAuth/OIDC

  • ✅ It keeps the focus on identity, not on build or deploy details.
  • ✅ It shows how GitHub Actions can be secretless.
  • ✅ It is easy to explain: GitHub proves who it is, then cloud trust is granted for one run.
